# Agent 训练功能指南

## 概述

OpenClaw IM 提供了完整的 Agent 训练体系，让你能够将任意 Agent 打造成特定领域的专家。训练方式灵活多样，支持**管理面板配置**和**聊天中渐进式训练**两种模式。

---

## 核心原理

OpenClaw 的 Agent 通过 **文件注入** 机制获取专业能力：

- 每个 Agent 拥有独立的工作空间目录（workspace）
- 目录中的 Markdown 文件在会话启动时自动注入到系统提示词中
- 关键文件：
  - `SOUL.md` — 角色定义与专长描述
  - `USER.md` — 用户偏好设置
  - `AGENTS.md` — 行为规则
  - 其他 `.md` 文件 — 领域知识文档

```
用户创建知识文件 → 文件存入 Agent workspace → Gateway 注入系统提示词 → LLM 基于知识回答
```

## 方式一：管理面板训练

### 进入训练面板

1. 在左侧「通讯录」或「虾聊」列表底部，点击 **「+ 添加 Agent」**
2. 进入 **Agent 管理面板**
3. 找到目标 Agent，点击右侧的 **⚙ 齿轮按钮**
4. 进入 **训练面板**（4 个 Tab 页）

### Tab 1：角色定义

- **快速模板**：内置多种专家模板（育儿专家、项目管理助手、代码审查专家、写作助手、数据分析师），选择后自动填充 SOUL.md
- **自定义编辑**：直接在大文本编辑器中编写或修改角色定义
- 点击 **「保存角色定义」** 生效

### Tab 2：知识库

- **查看文件**：列出 Agent workspace 中所有知识文件
- **新建文件**：点击 **「+ 新建文件」**，输入文件名（如 `feeding-guide.md`）
- **上传文件**：点击 **「上传文件」**，选择 `.md`/`.txt`/`.json` 文件
- **在线编辑**：点击文件名进入编辑器，修改后点击 **「保存」**
- **删除文件**：点击文件右侧的 **×** 按钮

### Tab 3：模型配置

- 以卡片形式展示所有可用模型
- 显示模型能力说明（中文能力、推理能力、上下文窗口大小等）
- 点击卡片选择模型，或勾选 **「使用系统默认模型」**
- 点击 **「保存模型配置」** 生效

### Tab 4：项目绑定

- 输入本地项目的完整路径（如 `C:\Projects\my-app`）
- Agent 的 workspace 将指向该目录
- Agent 可以读取项目中的文件来回答技术问题
- 点击 **「保存工作空间」** 生效

## 方式二：聊天中训练（渐进式）

在与 Agent 的 1v1 聊天中，可以边聊边训练，逐步丰富其知识。

### 训练工具栏

打开任意 Agent 的 1v1 聊天，聊天区顶部会自动显示 **训练工具栏**（仅主人可见），包含：

| 按钮 | 功能 |
|------|------|
| 📂 上传知识 | 选择 `.md`/`.txt`/`.json`/`.csv` 文件，直接保存到 Agent 知识库 |
| 💾 记住对话 | 将最近 20 条对话导出为 Markdown 文档，自动存入知识库 |
| ✏️ 编辑角色 | 快速跳转到训练面板，编辑 SOUL.md |

### 消息级操作

每条聊天消息的操作栏中，新增两类训练按钮：

- **📌 记住** — 将该消息内容保存到 Agent 知识库的 `notes.md` 中，支持渐进积累
- **📚 存入知识库** — 当消息包含文本文件附件（.md/.txt 等）时出现，点击将文件内容存入 Agent workspace

### 聊天指令

在聊天输入框中直接输入以下指令：

| 指令 | 说明 | 示例 |
|------|------|------|
| `/teach <内容>` | 将内容追加到 Agent 的 `notes.md` 知识文件 | `/teach 6个月以下婴儿不宜喝蜂蜜水` |
| `/remember` | 导出近期对话保存为知识文档 | `/remember` |
| `/soul <内容>` | 直接覆盖更新 Agent 的 SOUL.md 角色定义 | `/soul 你是一位资深育儿专家...` |

## 实战示例

### 示例 1：创建育儿专家

1. **创建 Agent**：管理面板 → 创建 Agent（ID: `parenting-expert`，名称: `育儿专家`）
2. **选择模板**：点击 ⚙ → 角色定义 Tab → 选择「育儿专家」模板
3. **添加知识**：知识库 Tab → 上传育儿文档（如 `feeding-guide.md`、`vaccine-schedule.md`）
4. **选择模型**：模型配置 Tab → 选择推理能力强的模型
5. **保存** → Agent 即成为育儿领域专家
6. **持续训练**：在日常聊天中使用 `/teach` 补充知识，点击 📌 记住有价值的回答

### 示例 2：创建项目管理助手

1. **创建 Agent**：管理面板 → 创建 Agent（ID: `project-helper`，名称: `项目助手`）
2. **绑定项目**：点击 ⚙ → 项目绑定 Tab → 输入项目路径 `C:\Projects\my-app`
3. **设定角色**：角色定义 Tab → 选择「项目管理助手」模板
4. **保存** → Agent 的 workspace 指向项目目录，可读取项目文件回答问题

### 示例 3：聊天中渐进训练

```
用户：/teach 我们团队的代码规范是使用 TypeScript 严格模式，所有函数必须有返回类型注解

系统提示：📝 知识已添加到 notes.md

用户：帮我审查一下这段代码有没有问题？
Agent：根据您的团队规范，这段代码存在以下问题...（基于刚学到的知识回答）

用户：/remember

系统提示：💾 对话已保存到知识库
```

## 知识文件最佳实践

1. **结构化内容**：使用 Markdown 标题分层组织知识，便于 Agent 检索
2. **具体明确**：避免模糊描述，给出具体的规则、数据和示例
3. **渐进积累**：通过日常聊天中的 `/teach` 和 📌 记住，持续丰富知识
4. **定期整理**：在知识库 Tab 中查看和整理已积累的知识文件
5. **模型匹配**：根据任务复杂度选择合适的模型（复杂推理选大模型，日常对话选快速模型）
