# P3.1 Agent 工作流技术方案

> 状态：规划中 | 优先级：P3  
> 创建时间：2026-03-14

## 1. 设计理念

### 核心原则：IM 原生，对话即工作流

不做复杂的节点拖拽编辑器（那是 Coze/Dify 的路线），而是做 **IM 范式的工作流**：

- **定义像聊天**：用对话式的步骤描述，不需要学习新语法
- **执行像群聊**：工作流运行时就像一个自动化的群聊，每个 Agent 依次发言
- **结果可见可控**：所有中间结果在聊天面板实时可见，随时可中断、修改、重跑

### 与竞品对比

| 特性 | Coze/Dify | 虾饺 IM 工作流 |
|------|-----------|---------------|
| 构建方式 | 节点拖拽画布 | 步骤列表 + 快速模板 |
| 学习成本 | 中等（理解节点概念） | 极低（选 Agent + 写提示词） |
| 执行可见性 | 日志面板 | 聊天对话（实时可见） |
| 中途干预 | 需要调试模式 | 直接在聊天中插话 |
| 构建速度 | 5-10 分钟 | 30 秒（模板）/ 2 分钟（自定义） |

## 2. 数据模型

### 2.1 工作流定义 (workflows.json)

```json
[
  {
    "id": "wf-content-pipeline",
    "name": "公众号生产线",
    "emoji": "📝",
    "description": "选题 → 写作 → 审核，一键生产公众号文章",
    "steps": [
      {
        "id": "s1",
        "name": "选题策划",
        "agent": "wechat-editor",
        "prompt": "请根据以下主题，策划3个公众号选题方案，每个方案包含标题、大纲、预期读者：\n\n{{input}}",
        "waitForApproval": false
      },
      {
        "id": "s2",
        "name": "文章撰写",
        "agent": "wechat-writer",
        "prompt": "请根据以下选题方案，选择最佳方案撰写一篇完整的公众号文章（2000字左右）：\n\n{{s1.output}}",
        "waitForApproval": true
      },
      {
        "id": "s3",
        "name": "审核优化",
        "agent": "wechat-publisher",
        "prompt": "请审核以下文章，给出修改建议并输出最终优化版：\n\n{{s2.output}}",
        "waitForApproval": false
      }
    ],
    "createdAt": 1710000000000
  }
]
```

### 2.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `steps[].agent` | string | 执行该步骤的 Agent ID |
| `steps[].prompt` | string | 提示词模板，支持变量插值 |
| `steps[].waitForApproval` | bool | 是否等待用户确认后再继续下一步 |

### 2.3 变量插值

| 变量 | 含义 |
|------|------|
| `{{input}}` | 用户触发工作流时的输入文本 |
| `{{s1.output}}` | 步骤 s1 的完整输出 |
| `{{sN.output}}` | 任意步骤的输出，按 step ID 引用 |
| `{{agent.name}}` | 当前步骤 Agent 的名称 |

## 3. 执行引擎

### 3.1 执行流程

```
用户触发: "运行「公众号生产线」，主题：AI编程"
  │
  ├─ 1. 创建执行实例 (runId)
  │     channel = "wf-{workflowId}-{timestamp}"
  │     state = { runId, stepIndex: 0, outputs: {}, status: 'running' }
  │
  ├─ 2. 执行 Step 1 (选题策划)
  │     ├─ 渲染 prompt: 替换 {{input}} → "AI编程"
  │     ├─ broadcast: 「📋 步骤 1/3: 选题策划 → @wechat-editor」
  │     ├─ sendToLLM(channel, "wechat-editor", renderedPrompt)
  │     ├─ 等待 Agent 回复 → 存入 outputs["s1"]
  │     └─ broadcast: 「✅ 步骤 1 完成」
  │
  ├─ 3. 检查 waitForApproval (Step 2)
  │     ├─ true → broadcast: 「⏸ 等待确认：是否继续写作？[继续] [跳过] [终止]」
  │     ├─ 用户点「继续」→ 执行 Step 2
  │     └─ 用户点「终止」→ 终止工作流
  │
  ├─ 4. 执行 Step 2 (文章撰写)
  │     ├─ 渲染 prompt: 替换 {{s1.output}}
  │     ├─ sendToLLM(channel, "wechat-writer", renderedPrompt)
  │     └─ ...
  │
  ├─ 5. 执行 Step 3 (审核优化)
  │     └─ ...
  │
  └─ 6. 完成
        broadcast: 「🎉 工作流完成！共 3 步，用时 2m 30s」
```

### 3.2 状态管理

```javascript
// 内存中的执行状态
activeWorkflows = Map<runId, {
  workflowId: string,
  channel: string,        // 专属聊天频道
  stepIndex: number,       // 当前步骤
  status: 'running' | 'waiting' | 'paused' | 'completed' | 'failed',
  input: string,           // 用户输入
  outputs: { [stepId]: string },  // 各步骤输出
  startTime: number,
  abortCtrl: AbortController
}>
```

### 3.3 与现有系统集成

| 集成点 | 方式 |
|--------|------|
| **消息发送** | 复用 `llm.sendToLLM(channel, agentId, prompt)` |
| **实时推送** | 复用 `gw.broadcast()` WebSocket 广播 |
| **中断机制** | 复用 `llm.cancelRun()` AbortController |
| **Agent 上下文** | 每步自动注入 Agent 的 SOUL.md + RAG 知识库 |
| **聊天频道** | 工作流创建专属 channel，消息进入 chat-history |

## 4. 前端交互

### 4.1 工作流入口

在通讯录的联系人列表中增加「工作流」分区：

```
通讯录
├── Agent (7)
│   ├── 🤖 公众号选题
│   ├── 🤖 公众号写作
│   └── ...
├── 群组 (1)
│   └── 👥 测试公众号流
└── 工作流 (2)          ← 新增
    ├── 📝 公众号生产线
    └── 📊 数据分析流程
    └── [+ 新建工作流]
```

### 4.2 快速构建器

点击「新建工作流」打开构建面板：

```
┌─────────────────────────────────────────────┐
│  新建工作流                            [×]   │
├─────────────────────────────────────────────┤
│                                             │
│  从模板创建：                                │
│  [📝 内容生产] [📊 数据分析] [🎨 创意设计]    │
│  [💬 客服接待] [📋 会议纪要] [空白工作流]     │
│                                             │
├─────────────────────────────────────────────┤
│  工作流名称: [公众号生产线          ]        │
│  描述:      [一键生产公众号文章      ]       │
│                                             │
│  步骤列表：                                  │
│  ┌─ Step 1 ──────────────────────────────┐  │
│  │ Agent: [🤖 公众号选题    ▾]           │  │
│  │ 任务:  [根据主题策划选题方案    ]      │  │
│  │ □ 执行后等待确认               [↕][×] │  │
│  └───────────────────────────────────────┘  │
│  ┌─ Step 2 ──────────────────────────────┐  │
│  │ Agent: [🤖 公众号写作    ▾]           │  │
│  │ 任务:  [根据选题撰写文章        ]      │  │
│  │ ☑ 执行后等待确认               [↕][×] │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  [+ 添加步骤]                                │
│                                             │
│  [保存工作流]                                │
└─────────────────────────────────────────────┘
```

关键 UX 设计：

- **任务描述自动生成 prompt**：用户只需写"根据主题策划选题方案"，系统自动拼接完整 prompt（含上一步输出注入）
- **拖拽排序**：步骤可上下拖拽调整顺序
- **从模板一键填充**：点击模板自动填入所有步骤

### 4.3 执行面板

点击工作流进入专属聊天频道：

```
┌─────────────────────────────────────────────┐
│  📝 公众号生产线                    ⚙ 编辑   │
│  选题(@选题) → 写作(@写作) → 审核(@审核)     │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─ 📋 Step 1/3: 选题策划 ───── ✅ 完成 ─┐ │
│  │  🤖 公众号选题:                         │ │
│  │  方案一：《AI编程入门指南》              │ │
│  │  方案二：《2026年AI工具横评》            │ │
│  │  方案三：...                            │ │
│  └─────────────────────────────────────────┘ │
│                                             │
│  ┌─ ✍️ Step 2/3: 文章撰写 ──── ⏳ 执行中 ─┐ │
│  │  🤖 公众号写作:                         │ │
│  │  # 2026年AI工具横评                     │ │
│  │  随着AI技术的飞速发展...█               │ │
│  └─────────────────────────────────────────┘ │
│                                             │
│  ── Step 3/3: 审核优化 ──── ⏳ 等待中 ──    │
│                                             │
├─────────────────────────────────────────────┤
│  [输入主题，开始工作流...]        [▶ 运行]  │
└─────────────────────────────────────────────┘
```

运行中状态：
- 发送按钮变为「■ 终止」（复用 P1.6 停止机制）
- 等待确认时显示「[继续] [跳过] [终止]」按钮
- 用户可在输入框中插话，作为额外指令注入当前步骤

### 5.1 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/workflows` | 列出所有工作流 |
| POST | `/api/workflows` | 创建工作流 |
| PUT | `/api/workflows/:id` | 更新工作流 |
| DELETE | `/api/workflows/:id` | 删除工作流 |
| POST | `/api/workflows/:id/run` | 触发执行，body: `{ input }` |
| POST | `/api/workflows/:id/runs/:runId/continue` | 用户确认继续 |
| POST | `/api/workflows/:id/runs/:runId/stop` | 终止执行 |
| GET | `/api/workflows/:id/runs` | 执行历史 |

### 5.2 WebSocket 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `workflow_step` | S→C | 步骤开始/完成/失败 |
| `workflow_waiting` | S→C | 等待用户确认 |
| `workflow_complete` | S→C | 工作流完成 |
| `workflow_control` | C→S | 用户操作：continue/skip/stop |
