# 虾饺 IM 语音会话技术方案

> 状态：待实现 | 优先级：中 | 预估工作量：3-5 天

## 目标

实现真正的语音对话体验，而非简单的语音转文字输入。用户可以通过语音与 Agent 进行连续对话，类似微信语音通话的交互方式。

## 方案对比

### 方案 A：模拟语音对话（推荐，中等复杂度）

本质是文字通道，但 UX 做成语音感觉。

**流程：**
1. 用户进入「语音对话模式」
2. 自动开启录音 → 浏览器 SpeechRecognition 识别为文字
3. 文字自动发送给 Agent
4. Agent 文字回复 → 自动调用 TTS 朗读
5. 朗读完毕 → 自动开启下一轮录音
6. 形成连续对话闭环

**技术栈：**
- ASR：浏览器原生 `SpeechRecognition` API（Chrome/Edge/Safari 支持中文）
- TTS：DashScope `qwen3-tts-flash`（需通用 API Key）或 CosyVoice
- 降级：浏览器 `SpeechSynthesis`（免费，质量一般）

**优点：**
- 架构改动小，复用现有消息通道
- 聊天记录天然有文字版本
- 可逐步迭代

**缺点：**
- 识别延迟（约 1-2 秒）
- 浏览器 SpeechRecognition 依赖网络（Chrome 走 Google 服务）

### 方案 B：端到端实时语音（高复杂度）

音频流直接走 WebSocket，不经过文字中间层。

**技术栈：**
- ASR：DashScope `paraformer-realtime-v2`（WebSocket 流式）
  - 端点：`wss://dashscope.aliyuncs.com/api-ws/v1/inference`
  - 支持实时流式识别，延迟低
- TTS：DashScope CosyVoice（WebSocket 流式）
  - 端点：`wss://dashscope.aliyuncs.com/api-ws/v1/inference`
  - 模型：`cosyvoice-v2` / `cosyvoice-v3-flash`
  - 音色：`longxiaochun` 等中文音色
- 浏览器端：`MediaRecorder` API 录音 + `AudioContext` 播放

**架构：**
```
浏览器 ──WebSocket──> IM Server ──WebSocket──> DashScope ASR
                                  ──WebSocket──> DashScope TTS
```

**优点：**
- 低延迟，接近实时
- 不依赖浏览器语音 API 兼容性

**缺点：**
- 架构改动大，需要管理多路 WebSocket
- 需要额外的 DashScope API Key（非 coding 子域）
- 聊天记录需要额外处理文字版本

## DashScope API 参考

### TTS（文本转语音）

**HTTP 端点（非流式）：**
```
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
Authorization: Bearer {API_KEY}
Content-Type: application/json

{
  "model": "qwen3-tts-flash",
  "input": {
    "text": "要合成的文本",
    "voice": "Cherry",
    "language_type": "Chinese"
  }
}
```

响应返回音频 URL（24 小时有效）：
```json
{
  "output": {
    "audio": {
      "url": "http://dashscope-result-bj.oss-cn-beijing.aliyuncs.com/...",
      "expires_at": 1766113409
    }
  }
}
```

**可用模型：**
- `qwen3-tts-flash` — 千问3 TTS，支持多语种
- `qwen3-tts-instruct-flash` — 支持指令控制语速语调

**可用音色：** Cherry 等（参见百炼文档）

### ASR（语音识别）

**实时 WebSocket：**
- 模型：`paraformer-realtime-v2`
- 端点：`wss://dashscope.aliyuncs.com/api-ws/v1/inference`
- 支持多语种（中文含方言、英文、日韩德法俄）

**录音文件识别（异步 REST）：**
- 端点：`POST https://dashscope.aliyuncs.com/api/v1/services/audio/asr/transcription`
- 需要提供公网可访问的音频 URL（不支持直传）
- 异步模式：提交任务 → 轮询结果

## API Key 要求

- 当前 `sk-sp-` 前缀的 Key 仅适用于 `coding.dashscope.aliyuncs.com`
- TTS/ASR 需要标准 `dashscope.aliyuncs.com` 域名的通用 API Key
- 在阿里云百炼控制台获取：https://bailian.console.aliyun.com/

## 前端 UI 设计要点

1. **语音对话模式入口**：输入框旁的麦克风按钮，点击进入全屏/半屏语音对话界面
2. **对话状态**：录音中（声波动画）→ 识别中 → Agent 思考中 → 播放回复中
3. **中断机制**：用户随时可以打断 Agent 朗读，开始新一轮输入
4. **退出**：点击挂断按钮或按 Esc 退出语音模式，回到文字聊天

## 实现顺序建议

1. 先实现方案 A 的基础版（浏览器 ASR + 浏览器 TTS）
2. 接入 DashScope TTS 提升音质（需通用 API Key）
3. 如有需要，升级到方案 B 的实时流式架构
